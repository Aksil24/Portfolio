📡 Dispositif & Méthode de veille

+

Afin de centraliser l'information de manière automatisée, j'ai mis en place un écosystème de surveillance structuré :

Fiche de veille — Vulnérabilité critique OpenSSH (regreSSHion)
Date de publication : 01/07/2024
Date de consultation : 15/09/2025
Auteur : Aksil OUKIL
Référence : CVE-2024-6387
Source : CERTFR-2024-ALE-009 ↗
Résumé
Une vulnérabilité critique a été découverte dans OpenSSH, baptisée regreSSHion. Elle permet à un attaquant non authentifié d'exécuter du code arbitraire à distance avec les privilèges root sur les serveurs Linux utilisant OpenSSH. C'est la première faille critique sur OpenSSH depuis 18 ans.
Systèmes / Produits concernés
  • OpenSSH versions 8.5p1 à 9.7p1
  • Toutes les distributions Linux utilisant glibc (Debian, Ubuntu, CentOS, etc.)
  • Non affecté : OpenBSD, Windows
Niveau de gravité
  • Score CVSS : 8.1 / 10
  • Criticité : Critique
  • Exploitation observée : Oui (preuve de concept publique)
Description technique
La faille est un race condition (situation de compétition) dans le gestionnaire de signal de sshd. En envoyant des connexions spécialement conçues au serveur SSH, un attaquant peut provoquer une corruption mémoire et exécuter du code en tant que root, sans aucune authentification nécessaire.
Impact potentiel
  • Prise de contrôle totale du serveur (privilèges root)
  • Vol de données stratégiques, injection de malwares, pivot latéral vers le réseau interne
  • Risque global extrême puisque SSH est exposé par défaut sur la quasi-totalité des machines Linux.
Impact pour une infrastructure SISR
  • Serveurs Linux en stage CECCA et en TP : tous les serveurs avec SSH exposé sont directement vulnérables.
  • Priorité : URGENTE — Planification d'une mise à jour immédiate.
Correctif / Mitigation recommandés
  • Mise à jour immédiate vers OpenSSH 9.8p1 ou supérieur.
  • En attendant l'application du correctif : restreindre le nombre de connexions simultanées via le paramètre MaxStartups dans sshd_config.
  • Restreindre les accès réseau SSH par adresse IP via le pare-feu applicatif (UFW, iptables).
  • Implémenter l'utilitaire Fail2ban pour freiner drastiquement les tentatives d'exploitation.
Lien avec MITRE ATT&CK
  • Tactique : Initial Access
  • Technique : T1190 — Exploit Public-Facing Application
Synthèse personnelle
« Cette faille m'a marqué car SSH est un service que j'utilise quotidiennement en TP et en stage. Ça montre qu'un service considéré comme intrinsèquement sûr peut contenir des failles critiques. Cela renforce l'importance de maintenir une politique stricte de mise à jour régulière, ce que j'ai pu mettre concrètement en pratique pendant mon stage CECCA lors de la phase de gestion des vulnérabilités. »
Fiche de veille — Élévation de privilèges AD (Zerologon)
Date de publication : 11/08/2020
Date de consultation : 10/01/2026
Auteur : Aksil OUKIL
Référence : CVE-2020-1472
Source : CERTFR-2020-ALE-020 ↗
Résumé
Zerologon est une vulnérabilité critique majeure découverte dans le protocole Netlogon de Windows Server. Elle permet à un attaquant distant ayant un simple accès réseau au contrôleur de domaine (DC) de devenir instantanément administrateur du domaine Active Directory sans posséder le moindre identifiant valide. Malgré les correctifs historiques, de nombreux systèmes obsolètes continuent d'être ciblés de manière active.
Systèmes / Produits concernés
  • Windows Server 2008 R2 jusqu'à Windows Server 2019
  • Tout contrôleur de domaine Active Directory dont les patchs de sécurité n'ont pas été appliqués.
Niveau de gravité
  • Score CVSS : 10.0 / 10 (Gravité maximale)
  • Criticité : CRITIQUE MAXIMUM
  • Exploitation observée : Oui (Largement exploitée par les groupes APT d'État et les affiliés de Ransomwares)
Description technique
Le protocole de chiffrement Netlogon s'appuie sur un mode de chiffrement AES-CFB8 mal implémenté, utilisant de façon erronée un vecteur d'initialisation (IV) statique fixé entièrement à zéro. En forçant l'envoi de requêtes d'authentification contenant uniquement des zéros, un attaquant possède 1 chance sur 256 de contourner le mécanisme de sécurité. Après succès, il est possible de modifier à la volée le mot de passe du compte machine du DC pour s'approprier le contrôle du domaine.
Impact potentiel
  • Compromission totale et irréversible de la forêt Active Directory
  • Accès sans restriction à tous les privilèges, identifiants de comptes et machines reliées
  • Déploiement massif et automatisé de rançongiciels sur l'intégralité du réseau d'entreprise
  • Exfiltration massive et compromission des données de l'entité
Impact pour une infrastructure SISR
  • L'Active Directory est l'élément centralisé au cœur du TP StadiumCompany (gestion fine des utilisateurs, application des stratégies GPO, configuration DNS).
  • Priorité : CRITIQUE — Audit impératif pour valider le niveau de patch de l'intégralité de nos contrôleurs de domaine (DC).
Correctif / Mitigation recommandés
  • Appliquer immédiatement le patch de sécurité Microsoft adéquat (KB4571702).
  • S'assurer du basculement forcé en mode "enforcement" de Netlogon (rendu obligatoire de base par Microsoft).
  • Mettre sous étroite surveillance les journaux d'événements Netlogon (analyse de l'Event ID 5829) pour intercepter d'éventuels comportements malveillants.
  • Segmenter et isoler logiquement les réseaux afin de cantonner et de limiter les flux directs à destination des contrôleurs de domaine.
Lien avec MITRE ATT&CK
  • Tactique : Privilege Escalation
  • Technique : T1068 — Exploitation for Privilege Escalation
Synthèse personnelle
« Zerologon est un cas d'école incontournable en cybersécurité : un score CVSS parfait de 10/10, combiné à une exploitation triviale et un impact dévastateur. Cette étude m'a permis de comprendre concrètement pourquoi les cycles de mise à jour de Windows Server ne doivent en aucun cas être reportés, surtout sur nos contrôleurs de domaine. En TP Active Directory, on se focalise souvent sur la configuration fonctionnelle, mais en conditions réelles, un unique DC non mis à jour peut faire s'effondrer la sécurité de tout le système d'information en l'espace de quelques secondes. »
